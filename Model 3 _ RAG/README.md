# Water Quality RAG Assistant

تطبيق Streamlit مبني على منطق النوتبوك بتاعك (`Untitled-1.ipynb`)، بيحوّله من نوتبوك تجريبي
إلى تطبيق ويب تفاعلي.

## هيكل المشروع (زي اللي عندك بالظبط)

```
project_root/
├── .env                <- ضع فيه GROQ_API_KEY (لا ترفعه على GitHub)
├── app.py              <- التطبيق (ضعه هنا في الجذر، بجانب فولدر documents)
├── documents/          <- ملفات الـ PDF المصدر
├── notebooks/          <- النوتبوك الأصلي (تقدر تسيبه فيه للتجارب فقط)
├── vector_db/           <- قاعدة بيانات Chroma (يتم إنشاؤها تلقائيًا أول مرة)
└── requirements.txt
```

## خطوات التشغيل

1. انسخ `app.py` و`requirements.txt` إلى **جذر المشروع** (بجانب فولدر `documents` و`vector_db`)
   -- وليس داخل فولدر `notebooks`.

2. فعّل الـ virtual environment الموجود عندك (`.venv`) ثم ثبّت المكتبات:
   ```bash
   pip install -r requirements.txt
   ```

3. أنشئ ملف `.env` في نفس المكان (انسخ من `.env.example`) وضع فيه مفتاح Groq الحقيقي (مجاني تمامًا، من console.groq.com):
   ```
   GROQ_API_KEY=مفتاحك_هنا
   ```

   ⚠️ **مهم جدًا:** المفتاح الموجود حاليًا داخل النوتبوك (خلية 23) مكتوب صراحةً في الكود.
   لا تكتب أي مفتاح API أبدًا مباشرة داخل الكود، واحصل على مفتاحك من console.groq.com
   هترفعه أو تشاركه.

4. شغّل التطبيق:
   ```bash
   streamlit run app.py
   ```

## طريقة عمل التطبيق

- أول مرة تشغّله، لو مفيش قاعدة بيانات في `vector_db/`، هيقرأ كل ملفات الـ PDF
  الموجودة في `documents/` تلقائيًا، يقسّمها، يحوّلها إلى embeddings، ويخزّنها.
- المرات اللي بعد كده، هيحمّل القاعدة الموجودة مباشرة (أسرع، بدون إعادة معالجة).
- من الشريط الجانبي تقدر:
  - ترفع ملفات PDF جديدة مباشرة من الواجهة.
  - تضغط "إعادة بناء قاعدة البيانات" لو غيّرت الملفات.
  - تكتب الـ API key يدويًا لو مش حابب تحطه في `.env`.
- في المحادثة، أي سؤال بتسأله بيتم البحث عن أقرب المقاطع في PDF ثم يتولّد الرد
  بواسطة Groq (نموذج Llama 3.3) بناءً على السياق فقط، مع عرض المصادر (اسم الملف ورقم الصفحة).
